# Skill Loading Map

Load only what's needed for the channel. This keeps context focused and prevents degradation on complex briefs.

## By Channel

| Channel | Load these skills | Load these references | Skip |
|---|---|---|---|
| **Email** | proven-email, proven-voice, proven-creative, proven-brand-qa | proven-creative: use only the reference for the needed photography category (1 file, not all 6) | proven-ui (unless building custom HTML beyond Klaviyo templates) |
| **Website / Landing page** | proven-ui, proven-voice, proven-creative, proven-brand-qa | proven-ui: all 3 refs. proven-creative: only the needed category ref | proven-email |
| **Product UI / Feature** | proven-ui, proven-brand-qa | proven-ui: tokens + frontend refs | proven-creative, proven-voice, proven-email (unless the feature needs copy or imagery) |
| **Social media** | proven-voice, proven-creative, proven-brand-qa | proven-creative: only the needed category ref | proven-ui, proven-email |
| **Paid media** | proven-voice, proven-creative, proven-brand-qa | proven-creative: only the needed category ref | proven-ui, proven-email |
| **Multi-channel** | All skills | Load refs selectively — one proven-creative category ref at a time as needed | — |

## Rules

1. **Never load all six proven-creative reference files at once.** Identify which photography category the brief needs (product catalog? skin? lifestyle?) and load only that one. If multiple categories are needed, load them sequentially — finish one image direction before loading the next.

2. **For multi-channel briefs:** complete one channel fully before moving to the next. Don't try to hold all channels in working memory simultaneously. The narrative and copy carry forward; the channel-specific assembly is done one at a time.

3. **proven-brand-qa is always loaded last**, after assembly. It's lightweight (~3,700 tokens) and only needed at the end.

4. **If context feels crowded** (responses getting shorter, details being dropped, instructions being skipped): stop, deliver what's complete, and suggest continuing the remaining channels in a follow-up session.
